## 🐛 Bug 列表

### P1 — 严重 (必须修复)

#### Bug #1: SetTotalDefeat 可被 SetFinalOutcome 覆盖
- **模块:** GameDirector
- **现象:** 调用 `SetTotalDefeat()` 后再调用 `SetFinalOutcome()`，结局被覆盖为 Defeat
- **影响:** 全军覆没场景可能被错误显示为普通失败
- **修复:** `SetFinalOutcome()` 开头增加 `if (Outcome == TotalDefeat) return;`

#### Bug #2: 不检测部队全灭
- **模块:** GameDirector
- **现象:** `Update()` 不调用 `AreAllFriendlyUnitsEliminated()`，所有部队被歼灭后游戏仍等到 09:00 才结束
- **影响:** 玩家在全灭状态下空等，体验极差
- **修复:** 在 `Update()` 中增加全灭检测，触发 `SetTotalDefeat()`

### P2 — 中等 (建议修复)

#### Bug #3: ApplyMoraleTick 缺少无通讯惩罚
- **模块:** BattleSimulator
- **现象:** `ApplyMoraleTick()` 只处理弹药不足(-5/tick)，GDD 要求的无通讯惩罚(-3/tick)需外部手动调用
- **修复:** 增加通讯状态追踪，超时自动应用惩罚

#### Bug #4: 重复注册累加初始兵力
- **模块:** BattleSimulator
- **现象:** 同 UnitID 重复调用 `RegisterFriendlyUnit()` 会累加 `InitialFriendlyTroops`，导致伤亡率计算错误
- **修复:** 注册前检查 ID 是否存在

#### Bug #5: 重大伤亡阈值过低
- **模块:** BattleSimulator
- **现象:** 阈值 10% 过于敏感，100 人部队每次受 11 点伤害就 -15 士气，几次交战就崩溃
- **修复:** 提高至 20-25%，或增加冷却时间

### P3 — 低 (可选修复)

#### Bug #6: 敌军弹药无消耗
- **模块:** BattleSimulator
- **现象:** 敌军弹药固定 100，无消耗逻辑
- **修复:** GDD 中明确为设计决策，或增加敌军弹药系统

#### Bug #7: 命令历史无限增长
- **模块:** CommandSystem
- **现象:** `commandHistory` 无上限，长游戏可能内存溢出
- **修复:** 添加最大历史记录限制 (200 条)
